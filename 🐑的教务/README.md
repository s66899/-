# 青羽教务系统使用说明书（重构版）

欢迎使用青羽教务系统重构版！本系统基于 Python Tkinter 开发，采用"校如云"架构设计，专为羽毛球培训机构设计，提供老师/管理员端和学生端双模式管理。本文档将逐步引导您完成安装、使用和数据维护。

---

## 一、系统简介

### 🏸 架构重构亮点
- **模仿"校如云"系统架构**：分为老师/管理员端和学生端，支持多角色访问
- **分层架构设计**：数据访问层、业务逻辑层、界面展示层分离
- **模块化功能组件**：学员管理、课程管理、排课消课、统计分析等独立模块
- **响应式界面**：现代化UI设计，支持PC端和移动端适配

### 🎯 核心功能模块

#### 老师/管理员端：
· **学员管理**：潜在/正式学员全生命周期管理，支持批量操作和CSV导入导出
· **课程管理**：课程与水平分离管理，课时跟踪，消耗率自动计算
· **排课管理**：智能排课系统，周日时间段自动修复，课表导入导出
· **消课点名**：一键点名考勤，剩余课时实时更新，存销比自动计算
· **统计中心**：实时运营数据看板，消课率统计，业绩分析报表

#### 学生端：
· **个人中心**：我的课表、在线约课、考勤签到、作业查看
· **在线服务**：请假申请、课程购买、师生互评、通知查看
· **消息通知**：上课提醒、续费通知、活动推送、课程调整

系统数据保存在本地 JSON 文件中，无需联网，安全可靠，支持定期备份恢复。

## 二、架构设计说明

### 1. 分层架构
```
┌─────────────────┐
│   界面展示层    │  ← 老师端/学生端界面
├─────────────────┤
│   业务逻辑层    │  ← 学员服务、课程服务、考勤服务
├─────────────────┤
│   数据访问层    │  ← 数据管理器(DataManager)
├─────────────────┤
│   数据存储层    │  ← JSON文件数据库
└─────────────────┘
```

### 2. 多角色支持
- **管理员**：完整系统管理权限
- **老师**：教学管理相关功能
- **学生**：个人学习和约课功能
- **家长**：学员信息查看和通知接收（开发中）
- **教练**：课程教学和学员管理（开发中）

### 3. 数据模型
- **学员(Student)**：姓名、电话、出生日期、等级、教练、状态
- **课程(Course)**：名称、水平、总课时、剩余课时、价格、教练
- **排课(Schedule)**：学员、课程、星期、时间段、状态
- **考勤(Attendance)**：学员、课程、排课、日期、状态、课时消耗

### 4. 技术特性
- **前后端分离**：Python后端 + Tkinter前端
- **模块化设计**：各功能模块独立，便于维护扩展
- **数据安全**：本地存储，支持加密备份
- **兼容性**：支持Windows/macOS/Linux系统

## 四、功能模块详解

### 1. 学员管理

· 添加学员：点击"添加"，弹出窗口填写姓名、电话、出生日期（或年龄）、等级、教练等信息。年龄将根据出生日期自动计算。
· 删除学员：选中列表中的学员（可多选），点击"删除"。支持全选勾选框进行批量删除。
· 全局搜索：在搜索框输入姓名或电话，实时过滤显示结果。
· 导入/导出：
 · 导入：从 CSV 文件批量添加学员（格式见后文）。
 · 导出：将当前列表（含搜索过滤后的结果）导出为 CSV 文件。
· 年龄显示：列表自动展示学员的周岁年龄。

### 2. 报课管理

· 课程与水平分离：每个报课记录关联一个"课程名称"和"学员水平"（如初级、中级），便于区分不同班级。
· 课时跟踪：显示总课时和剩余课时。每次消课后，剩余课时自动减少。
· 添加报课：选择学员，填写课程、水平、总课时，系统自动将剩余课时初始化为总课时。
· 批量导入：通过 CSV 文件批量生成报课记录（格式见后文）。

### 3. 排课管理

· 时间段管理：为每个星期（周一至周日）预设可上课的时间段，支持增、删、改。
· 课表导入：通过 CSV 文件批量导入学员排课（格式见后文）。导入时自动匹配学员和时段。
· 周日时间段修复：若周日时间段缺失，点击"修复周日"按钮，系统将自动复制周六的时间段到周日，避免排课遗漏。

### 4. 消课点名

· 点名消课：选择某次排课，勾选已到课的学员，点击"消课"按钮，对应学员的剩余课时减少 1（或指定课时数）。
· 剩余课时显示：消课界面实时展示每位学员的剩余课时，避免超扣。
· 存销比自动更新：每次消课后，系统自动重新计算该学员的存销比（已消耗课时 ÷ 总课时），并更新到统计中心。

### 5. 统计中心

· 实时数据看板：展示学员总数、活跃学员数、总课时数、总消耗课时等关键指标。
· 存销比计算：根据每位学员的消课记录，计算整体存销比，帮助判断课程消耗进度。
· 图表展示（如有）：可直观对比各课程或各教练的消课情况。

## 六、数据导入导出格式

所有 CSV 文件建议使用 UTF-8 编码，第一行为列名。

### 1. 学员导入（CSV）

列名：`姓名,电话,年龄,等级,教练`

示例：

```
姓名,电话,年龄,等级,教练
张三,13800000000,8,初级,李教练
李四,13900000000,10,中级,王教练
```
年龄字段可以是整数或出生日期（YYYY-MM-DD），系统会自动处理。

### 2. 报课导入（CSV）

列名：`姓名,电话,课程,水平,总课时,剩余课时`

示例：

```
姓名,电话,课程,水平,总课时,剩余课时
张三,13800000000,羽毛球基础班,初级,20,20
李四,13900000000,羽毛球提高班,中级,30,25
```
系统会根据"姓名+电话"匹配学员，若不存在则会提示先添加学员。

### 3. 课表导入（CSV）

列名：`姓名,电话,星期,时间段`

示例：

```
姓名,电话,星期,时间段
张三,13800000000,星期一,16:00-18:00
李四,13900000000,星期三,18:00-20:00
```
· 星期：使用中文（星期一、星期二……星期日）或数字 1~7。
· 时间段：格式必须与系统中预设的时间段完全一致（如"16:00-18:00"），否则无法匹配。

## 七、常见问题与注意事项

**Q1：导入学员或报课时提示"找不到学员"？**  
A：请确保导入的学员已在"学员管理"中录入。系统会按"姓名+电话"组合匹配，二者必须完全一致。

**Q2：消课后剩余课时变成负数怎么办？**  
A：系统默认不会允许消课超过剩余课时，若出现负数请检查是否重复消课或数据错误。可手动调整报课记录中的剩余课时。

**Q3：周日时间段修复后，之前的数据会丢失吗？**  
A：不会。修复仅复制周六的时间段到周日，不会覆盖已有周日排课数据。若周日已有排课，请先确认后再操作。

**Q4：如何备份数据？**  
A：直接复制 `教务数据.json` 文件到其他位置即可。恢复时覆盖原文件。

**Q5：系统窗口显示不全或字体太小？**  
A：可在代码中调整窗口初始大小或字体设置，建议屏幕分辨率不低于 1280×720。
